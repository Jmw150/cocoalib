package    --    keyword marking content of a CoCoA package


Syntax 
package PkgName: STRING);

Description
This is for more advanced use of CoCoA-5, when you have several functions for closely related operations.

We recommend putting all functions belonging together in a package into a single file (whose name should indicate the purpose of the package). Inside the file, at the start write package followed by the package name (usu. the same as as the file name); at the end of the file put EndPackage;.

See the entry First Example of a Package .

Example 
/**/ package $demo;
/**/   export MyFunc;
/**/   define MyFunc(N) return N+1; enddefine;
/**/ EndPackage;
